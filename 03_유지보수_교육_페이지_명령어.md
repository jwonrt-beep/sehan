# 세한메카트로닉스 유지보수·교육 페이지 전용 명령어

> "시공 후에도 책임진다"는 신뢰를 구축하는 페이지용 명령어

---

## 1. 공통 원칙

- **강의**가 아니라 **현장 운영 가능성** 강조
- "우리가 끝까지 책임진다"는 메시지 일관 유지
- 복잡한 설명보다 **프로세스·기준·대응 방식** 명확화

---

# Part A: 유지보수 페이지

---

## 2. 유지보수 페이지 역할

- 고객 불안 "고장 시 누가, 얼마나 빨리?" 해소
- 별도 메뉴로 구성 (메인 네비게이션에 노출)

---

## 3. 필수 포함 항목 (4가지)

### 3-1. 대응 프로세스

| 항목 | 명령어 |
|------|--------|
| 제목 | "대응 프로세스" / "고장 발생 시" |
| 필수 포함 | 문의 접수 → 현황 파악 → 현장 출동/원격 → 조치 완료 흐름 |
| 표현 | 플로우차트 또는 단계별 번호 리스트 |
| 톤 | "몇 단계 안에" 등 구체적 시간 언급 가능 시 포함 |

**예시:**  
"1) 고장 접수 → 2) 원격 진단 또는 현장 방문 결정 → 3) 24~48시간 내 현장 출동 (지역별 상이)"

---

### 3-2. 현장 방문 기준

| 항목 | 명령어 |
|------|--------|
| 제목 | "현장 방문 기준" / "출동 기준" |
| 필수 포함 | 어떤 경우 현장 출동하는지, 지역별 소요 시간 범위 |
| 톤 | 과장 없이, 시흥·천안 등 거점 기준 현실적으로 |

**예시:**  
"수도권: 접수 후 24~48시간 내 출동. 지방: 48~72시간. 긴급 고장은 별도 협의."

---

### 3-3. 소모품·부품 대응 방식

| 항목 | 명령어 |
|------|--------|
| 제목 | "소모품·부품" / "부품 공급" |
| 필수 포함 | 토치 노즐, 와이어, 가스 등 소모품 공급 경로, 부품 수급 기간 |
| 톤 | 고객이 미리 준비할 수 있는 정보 제공 |

**예시:**  
"표준 소모품은 재고 보유. 특수 부품은 발주 후 1~2주. 정기 점검 시 소모품 확인 포함."

---

### 3-4. 예방 점검 가능 여부

| 항목 | 명령어 |
|------|--------|
| 제목 | "예방 점검" / "정기 점검" |
| 필수 포함 | 예방 점검 제공 여부, 주기, 점검 항목 개요 |
| 톤 | "사고 전에 막는다"는 안심 메시지 |

**예시:**  
"분기별 또는 반기별 예방 점검 가능. 용접 토치, 케이블, 가스 라인, 전극 마모 등 확인."

---

## 4. 유지보수 페이지 구성 순서

1. 대응 프로세스  
2. 현장 방문 기준  
3. 소모품·부품 대응 방식  
4. 예방 점검 가능 여부  
5. (선택) 유지보수 문의/계약 안내

---

## 5. 유지보수 금지 사항

- [ ] "언제든 빠르게" 등 모호한 표현
- [ ] 현실 불가능한 출동 시간 약속
- [ ] 부품·소모품 공급에 대한 불명확한 서술

---

# Part B: 교육 페이지

---

## 6. 교육 페이지 역할

- "강의"가 아니라 **현장 운영 가능성** 강조
- 작업자가 **실제로 로봇을 운영·관리할 수 있게** 하는 것

---

## 7. 교육 관점 3분류 (명확히 구분)

### 7-1. 작업자 교육

| 항목 | 명령어 |
|------|--------|
| 대상 | 현장 작업자 |
| 목표 | 일상 기동·정지, 간단 조작, 이상 감지 시 대응 |
| 포함 내용 | 조작 패널 사용법, 안전 수칙, 비상 정지, 일상 점검 항목 |
| 톤 | "강의"가 아니라 " hands-on, 현장에서 바로 쓰는" |

---

### 7-2. 관리자 교육

| 항목 | 명령어 |
|------|--------|
| 대상 | 생산·설비 관리자 |
| 목표 | 생산 계획 연동, 성능 모니터링, 유지보수 의뢰 판단 |
| 포함 내용 | 데이터 확인, 알람 대응, 세한 연락 시점 판단 |
| 톤 | "문제 발생 전 파악" 가능하도록 |

---

### 7-3. 기술자 교육

| 항목 | 명령어 |
|------|--------|
| 대상 | 설비·기술 담당자 |
| 목표 | 용접 조건 조정, 간단 프로그램 수정, 고장 원인 추적 |
| 포함 내용 | 파라미터 조정, 용접 조건 최적화, 로그 분석 기초 |
| 톤 | 세한 없이 일부 문제 자체 해결 가능 수준 |

---

## 8. 교육 페이지 필수 포함

| 항목 | 명령어 |
|------|--------|
| 교육 범위 | 작업자/관리자/기술자 각각 무엇을 배우는지 |
| 교육 형태 | 현장 교육 / 세한 사옥 / 온라인 등 |
| 교육 일정 | 도입 시 기본 교육 포함 여부, 추가 교육 가능 여부 |
| 인수인계 | 시운전 완료 시 점검표·매뉴얼 전달 등 |

---

## 9. 교육 금지 사항

- [ ] "강의", "교육 과정" 중심 표현
- [ ] 작업자·관리자·기술자 구분 없이 통짜 서술
- [ ] "배우면 다 된다"는 과장

---

## 10. 유지보수·교육 페이지 톤 통일

- 기술자는 **신뢰**할 수 있게 (프로세스·기준 명확)
- 구매 담당자는 **이해**하기 쉽게 (복잡한 용어 최소화)
- 대표는 **리스크 감소**를 느끼게 (예방 점검, 인수인계 강조)
